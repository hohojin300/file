1.버전 관리 도구(git=DVCS)
  -변경 이력을 확인하는 작업
  -공동 작업할때 개발자가 행한 변경 사항을 합치는 작업
  -변경 사항을 합칠 때 충돌이 된 상황=>문제를 해결하는 작업
  -특정 시점의 버전과 비교,특정 시점의 버전을 가져오는 작업

2.버전 관리 저장소(repository)
    -변경 이력 저장
    -원격저장소 : 자신의 변경내용을 반영,
                다른 개발자의 변경 내용을 가져오는 장소
    -지역저장소 : 개발이 이루어지는 장소

3.버전 관리의 시작
  -저장소 : 변경 이력의 저장 장소 (git init) 
  -버전관리의 시작 : (지역) 저장소 만들기로 시작함
  -저장소가 만들어지면 : 파일들의 변경 내역이 추적됨
